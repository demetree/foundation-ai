containerd is the low-level container runtime that actually starts, stops, and supervises individual container processes on a Linux host. Originally extracted from the Docker Engine in 2017 to make the underlying machinery available to other higher-level tools, it has since become a graduated project at the Cloud Native Computing Foundation and serves as the default runtime in most modern Kubernetes installations. The project deliberately occupies a narrow scope: it pulls and stores images, manages container lifecycles, configures networks via plugins, and exposes a stable gRPC API -- but it leaves higher-level concerns like build tooling, orchestration, and developer ergonomics to the layers above it. This focus on a small, well-tested core that other tools can compose against is part of why it ended up underneath Docker, Kubernetes, and several other competing container platforms simultaneously.
