Redis is an in-memory data structure server first released in 2009. Unlike a traditional database that stores rows in tables on disk, it keeps its working set entirely in RAM and exposes a small, sharp set of native operations on rich data types -- strings, hashes, sorted sets, streams, hyperloglogs, geo-indexed points -- through a simple text protocol. Sub-millisecond latency at very high throughput on commodity hardware is its hallmark, which is why it has come to dominate the caching tier of essentially every modern web architecture and is also widely used as a transient message broker, a session store, a real-time leaderboard, and a rate-limiting backplane. Persistence is optional: the user can pick between point-in-time RDB snapshots, an append-only log of mutating commands, or both, depending on the durability budget of the workload. Cluster mode shards keys across nodes by hash slot, and a replica protocol provides synchronous or asynchronous read scaling.
