llama.cpp is a from-scratch C++ implementation of transformer inference that started life as a port of LLaMA to a single laptop, and has since grown into the de facto runtime for quantised open-weight models on consumer hardware. Its central technical contribution is a custom file format called GGUF that stores model weights in low-bit-width quantised form -- 2, 3, 4, 5, 6, or 8 bits per parameter -- so that a 7-billion-parameter network that would normally need 14 GB of memory at full precision can fit in 4 GB and still produce useful output. The project includes hand-tuned compute kernels for x86 SIMD, ARM NEON, Apple's Accelerate framework, CUDA, Vulkan, and SYCL, which means it runs reasonably on essentially every modern device including phones. A vibrant ecosystem of bindings has grown up around it for Python, C#, Rust, Go, and JavaScript.
